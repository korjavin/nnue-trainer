# gobot_search_parity.jsonl — GoBot fixed-depth move-parity oracle

Generated by `virusgame/backend/arena/cmd/searchparitygen` (Phase 1, bd nnue-trainer-0dj.2).
Each line is one deterministic `search.ChooseDepth(ctx, state, depth)` result from GoBot.
A faithful Java port of the GoBot search must, for every record, pick the **same
action** and return the **same integer score** at the given `depth`.

## Record schema

```json
{
  "board": [[{"owner":int,"kind":str}, ...], ...],
  "player": int,          // == state.CurrentPlayer(); the mover / search root
  "depth": int,           // fixed search depth (3 or 5 in this fixture)
  "score": int,           // Result.Score from ChooseDepth (leaf = HandTunedEval)
  "action": { ... },      // chosen Result.Action, encoded below
  "movesLeft": int,       // hidden state — feeds tempo terms AND stateHash
  "neutralUsed": [bool]   // hidden state, index = player-1 — feeds neutral bonus AND stateHash
}
```

- `owner`: 0 neutral, 1/2 players. `kind`: `EMPTY|NORMAL|FORTIFIED|NEUTRAL|BASE`.
- Board is the exact form `HandTunedEvalParityTest.toBoard` already parses (same
  cell encoding as `gobot_staticeval_parity.jsonl`).

## Action encoding (matches Java `MoveAction` / `PlaceNeutralsAction`)

```json
"action": {"type":"MOVE","target":{"row":r,"col":c}}
"action": {"type":"PLACE_NEUTRALS","pos1":{"row":r,"col":c},"pos2":{"row":r,"col":c}}
```

`pos1`/`pos2` are Go `Action.Neutrals[0]`/`[1]`. `PlaceNeutralsAction.equals` is
order-independent, so the parity test may compare the pair as an unordered set.

## Hidden State the port MUST reproduce

`ChooseDepth` does **not** consult the opening book (only `Choose`/`ChooseNodeBudget`
do), so every record is pure search from the given position. The search decision and
`stateHash` (TT key) depend on non-board state that the board grid cannot encode:

- `player` = `state.CurrentPlayer()` — the maximizing root.
- `movesLeft` = actions remaining this turn; enters `HandTunedEval` tempo terms and
  `stateHash` (`add(byte(state.MovesLeft()))`).
- `neutralUsed[p-1]` = whether player `p` has spent its neutral placement; enters the
  per-player neutral bonus and `stateHash` (`add(byte(player)|0x20)`).

The Java port must build its position from `board + player + movesLeft + neutralUsed`
(same lesson as Phase 0's `StaticEval`) or move choices/scores will diverge.

## Regenerating

```bash
cd ../virusgame/backend
go build -o /tmp/searchparitygen ./arena/cmd/searchparitygen/
/tmp/searchparitygen -out .../src/test/resources/gobot_search_parity.jsonl \
    -positions 400 -sample 3 -timeout 800 -seed 1
```

`-timeout` only gates **selection**: a pathologically wide position (many
neutral-placement pairs) whose depth-5 search would blow up is skipped. Every kept
record is a fully-completed, deterministic search — reproducible in Java with no timeout.
This fixture: 412 records, depths {3,5}, players {1,2}, both action types.
