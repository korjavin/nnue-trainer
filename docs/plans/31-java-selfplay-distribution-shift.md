# Implementation Plan - Fix Distribution Shift via Java-Distribution Self-Play Pipeline (nnue-trainer-ntd.5.4)

## Issue
`nnue-trainer-ntd.5.4`: Lever 2: Java-distribution self-play -> retrain -> eval (fix distribution shift).

## Problem Context
The NNUE model trained on GoBot-vs-GoBot self-play games exhibits a distribution shift when queried on positions generated by the Java engine. Generating self-play training games directly using Java `SearchEngine` instances resolves this distribution mismatch. Self-play games must include opening move randomization/exploration to ensure diverse dataset generation (preventing collapse to identical repetitive game trees).

## Design Goals
1. Enhance `SelfPlayGenerator.java` to support configurable search budgets (`depth` or `timeLimitMs`) and epsilon-random exploration (e.g. 10% chance of random legal move in early plies).
2. Integrate `SelfPlayGenerator` with `PeriodicRetrainer.java` so periodic self-play generation outputs balanced datasets into the retraining queue.
3. Add distinct-game metrics calculation to `SelfPlayGenerator` (ratio of unique board configurations / hash counts).
4. Add unit test coverage in `SelfPlayGeneratorTest.java` verifying game diversity and record export format.

## Step-by-Step Implementation

### Step 1: Update `SelfPlayGenerator.java`
- Add `epsilon` exploration parameter (`default 0.1`) for early game plies (turns <= 6).
- Implement unique position hash counter to compute and log `distinctGameRatio`.
- Support configurable search depth (`default 2`) or time limit.

### Step 2: Update `PeriodicRetrainer.java`
- Connect `SelfPlayGenerator` outputs to trigger retraining when new self-play datasets are generated.

### Step 3: Tests
- Update `SelfPlayGeneratorTest.java` to assert `distinctGameRatio > 0.8` when self-play generates games with epsilon exploration.

## Verification Command
```bash
./mvnw test -Dtest=SelfPlayGeneratorTest,PeriodicRetrainerTest
```
